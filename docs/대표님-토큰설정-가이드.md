# SocialDoctors 플랫폼 토큰 설정 가이드

> 대표님이 직접 진행해주셔야 하는 작업 목록입니다.
> 총 소요시간: 약 1~2시간 (TikTok 심사 제외)

---

## 1단계: Meta (Facebook + Instagram + Threads) — 15분

### 사전 준비
- Facebook 계정 (대표님 개인 또는 회사)
- Facebook 페이지 (홍보할 페이지)
- Instagram 비즈니스 계정 (설정 → 계정 → 비즈니스 계정으로 전환)

### 절차

#### 1-1. Meta 개발자 등록
1. https://developers.facebook.com 접속
2. 우측 상단 "시작하기" 클릭
3. Facebook 계정으로 로그인
4. 개발자 약관 동의

#### 1-2. 앱 생성
1. "내 앱" → "앱 만들기" 클릭
2. 앱 유형: "비즈니스" 선택
3. 앱 이름: "SocialDoctors" 입력
4. 앱 연락처 이메일 입력
5. "앱 만들기" 클릭

#### 1-3. Page Access Token 발급
1. https://developers.facebook.com/tools/explorer/ 접속 (Graph API Explorer)
2. 우측 상단 앱 선택: "SocialDoctors"
3. "Generate Access Token" 클릭
4. 요청 권한:
   - `pages_manage_posts` (페이지 포스팅)
   - `pages_read_engagement` (인사이트)
   - `instagram_basic` (IG 기본)
   - `instagram_content_publish` (IG 발행)
   - `threads_basic` (Threads 기본)
   - `threads_content_publish` (Threads 발행)
5. Facebook 로그인 → 권한 승인 → 페이지 선택
6. 생성된 토큰 복사 (User Access Token)

#### 1-4. Long-Lived Token으로 변환 (60일 유효)
Graph API Explorer에서:
```
GET /oauth/access_token?grant_type=fb_exchange_token&client_id={앱ID}&client_secret={앱시크릿}&fb_exchange_token={방금복사한토큰}
```
응답의 `access_token`이 60일짜리 Long-Lived Token입니다.

#### 1-5. Page Access Token 획득
```
GET /me/accounts?access_token={Long-Lived-Token}
```
응답에서 해당 페이지의 `access_token`이 영구 Page Token입니다.

#### 1-6. 저장할 값
```
META_APP_ID=앱 대시보드 → 설정 → 기본 → 앱 ID
META_APP_SECRET=앱 대시보드 → 설정 → 기본 → 앱 시크릿
```

#### 1-7. SocialDoctors 채널 등록
Admin → Social Pulse → 채널 추가:
- Facebook: pageId = 페이지 ID, accessToken = Page Token
- Instagram: pageId = IG 비즈니스 계정 ID, accessToken = 위와 동일 토큰
- Threads: pageId = Threads 사용자 ID, accessToken = 위와 동일 토큰

## 2단계: X (Twitter) — 30분

### 절차

#### 2-1. 개발자 포털 가입
1. https://developer.x.com 접속
2. X(Twitter) 계정으로 로그인
3. "Sign up for Free Account" 클릭
4. 사용 목적 250자 이상 작성:
   ```
   SocialDoctors는 비즈니스 홍보 자동화 플랫폼입니다. 
   외부 프로젝트의 홍보를 대행하여 X에 마케팅 콘텐츠를 
   자동 발행합니다. 텍스트 기반 마케팅 포스트와 
   프로모션 정보를 게시하며, 사용자 데이터는 수집하지 않습니다.
   ```
5. 약관 동의 → 제출

#### 2-2. 프로젝트 + 앱 생성
1. Dashboard → "+ Create Project" 
2. 프로젝트 이름: "SocialDoctors"
3. 사용 사례: "Making a bot" 선택
4. 앱 이름: "SD-Publisher"

#### 2-3. 키 발급
프로젝트 → 앱 → "Keys and tokens" 탭:
1. API Key and Secret → "Regenerate" → 복사
2. Bearer Token → "Generate" → 복사
3. Access Token and Secret → "Generate" → 복사

#### 2-4. 저장할 값
```
X_API_KEY=Consumer Keys → API Key
X_API_SECRET=Consumer Keys → API Key Secret
X_BEARER_TOKEN=Authentication Tokens → Bearer Token
X_ACCESS_TOKEN=Authentication Tokens → Access Token
X_ACCESS_SECRET=Authentication Tokens → Access Token Secret
```

#### 2-5. SocialDoctors 채널 등록
Admin → Social Pulse → 채널 추가:
- Platform: X
- pageId: X 사용자 ID (settings → account → account information)
- accessToken: Bearer Token

## 3단계: YouTube — 1~2시간

### 절차

#### 3-1. Google Cloud 프로젝트 생성
1. https://console.cloud.google.com 접속
2. 새 프로젝트 생성: "SocialDoctors"
3. 프로젝트 선택

#### 3-2. YouTube Data API v3 활성화
1. "API 및 서비스" → "라이브러리"
2. "YouTube Data API v3" 검색 → "사용" 클릭

#### 3-3. OAuth 동의화면 설정
1. "API 및 서비스" → "OAuth 동의 화면"
2. 사용자 유형: "외부" 선택
3. 앱 이름: "SocialDoctors"
4. 사용자 지원 이메일: 대표님 이메일
5. 범위 추가:
   - `https://www.googleapis.com/auth/youtube.upload`
   - `https://www.googleapis.com/auth/youtube.readonly`
6. 테스트 사용자에 대표님 Gmail 추가

#### 3-4. OAuth 클라이언트 ID 생성
1. "API 및 서비스" → "사용자 인증 정보"
2. "사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"
3. 애플리케이션 유형: "웹 애플리케이션"
4. 리다이렉트 URI: `https://app.socialdoctors.kr/api/auth/callback/google`
5. 생성 → 클라이언트 ID + 시크릿 복사

#### 3-5. Refresh Token 발급
브라우저에서 아래 URL 접속 (클라이언트ID 교체):
```
https://accounts.google.com/o/oauth2/v2/auth?
  client_id={클라이언트ID}&
  redirect_uri=https://app.socialdoctors.kr/api/auth/callback/google&
  response_type=code&
  scope=https://www.googleapis.com/auth/youtube.upload&
  access_type=offline&
  prompt=consent
```
승인 후 리다이렉트된 URL에서 `code` 파라미터 복사.

터미널에서:
```bash
curl -X POST https://oauth2.googleapis.com/token \
  -d "code={코드}&client_id={클라이언트ID}&client_secret={시크릿}&redirect_uri=https://app.socialdoctors.kr/api/auth/callback/google&grant_type=authorization_code"
```
응답의 `refresh_token` 저장.

#### 3-6. 저장할 값
```
YOUTUBE_CLIENT_ID=OAuth 클라이언트 ID
YOUTUBE_CLIENT_SECRET=OAuth 클라이언트 시크릿
YOUTUBE_REFRESH_TOKEN=위에서 발급한 refresh_token
```

## 4단계: TikTok — 5~10 영업일 (심사)

### 절차

#### 4-1. 개발자 등록
1. https://developers.tiktok.com 접속
2. TikTok 계정으로 로그인
3. "Manage Apps" → "Create App"

#### 4-2. 앱 설정
1. 앱 이름: "SocialDoctors Publisher"
2. Platform: Web
3. Products: "Content Posting API" 선택
4. Scopes: `video.publish`, `video.upload`
5. Privacy Policy URL: `https://socialdoctors.kr/privacy`
6. Website URL: `https://socialdoctors.kr`

#### 4-3. 샌드박스 테스트
1. 개발자 포털에서 샌드박스 환경 접속
2. 테스트 영상을 업로드하여 발행 테스트
3. 성공 확인 후 화면 녹화 (데모 영상)

#### 4-4. 프로덕션 심사 제출
1. "Submit for Review" 클릭
2. 데모 영상 업로드 (30초~2분)
3. 사용 설명 작성
4. 심사 대기 (5~10 영업일)

#### 4-5. 저장할 값 (심사 승인 후)
```
TIKTOK_CLIENT_KEY=앱 설정 → Client Key
TIKTOK_CLIENT_SECRET=앱 설정 → Client Secret
```
