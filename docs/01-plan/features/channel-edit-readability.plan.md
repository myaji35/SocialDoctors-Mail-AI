# Plan: channel-edit-readability

## 개요
- **기능명**: 채널 정보 수정 모달 가독성 개선
- **유형**: UX 개선 (Readability)
- **우선순위**: 높음
- **예상 작업량**: 소규모 (1개 파일)

## 현재 문제점 분석

### 1. 읽기 전용 정보 영역 (160~183줄)
- **문제**: `bg-gray-50` 배경 + `text-gray-500` 라벨 → 대비 부족
- **문제**: 5개 항목이 빽빽하게 나열 (`space-y-2.5`) → 행 간격 부족
- **문제**: `text-sm` 일괄 적용 → 계층 구분 어려움

### 2. 편집 필드 영역 (186~280줄)
- **문제**: "편집 가능 필드" 섹션 헤더가 회색 텍스트(`text-gray-700`)로 눈에 띄지 않음
- **문제**: 라벨(`text-sm font-medium text-gray-700`)과 인풋 사이의 시각적 구분 약함
- **문제**: 토큰 textarea가 `text-sm` + 마스킹(`disc`) → 입력 내용 판독 어려움

### 3. 헤더 영역 (146줄)
- **문제**: "채널 정보 수정" 타이틀에 어떤 채널인지 정보 없음
- **문제**: 플랫폼 아이콘 없음 → 채널 컨텍스트 인지 어려움

### 4. 전반적 레이아웃
- **문제**: 읽기 전용 + 편집 가능 섹션의 시각적 분리가 `space-y-5`만으로 약함
- **문제**: 액션 버튼 영역과 Danger Zone 사이 구분 약함

## 개선 사항

### A. 헤더 개선
- 플랫폼 아이콘 + 채널명을 헤더에 표시 → "Facebook · 공식 페이지" 형태
- 현재 상태 뱃지(활성/비활성)를 헤더에 이동

### B. 읽기 전용 정보 영역 가독성
- 라벨 색상 대비 강화: `text-gray-500` → `text-gray-600`
- 행 간격 확대: `space-y-2.5` → `space-y-3` + `py-1` 패딩 추가
- 값 텍스트 크기/굵기 강화: `font-medium text-gray-800` → `font-semibold text-gray-900`
- 구분선 추가: 항목 사이 subtle divider

### C. 편집 필드 가독성
- 섹션 헤더 강화: 아이콘 추가 + `text-gray-900 font-bold` + 배경 구분
- 라벨과 인풋 간격: `mb-1` → `mb-2`
- 인풋 높이 확대: `py-2.5` → `py-3`
- 토큰 textarea: font-size 유지하되 `leading-relaxed` 추가
- 보조 텍스트: `text-gray-400` → `text-gray-500` (대비 강화)

### D. 액션 버튼 영역
- 버튼 높이 확대: `py-2.5` → `py-3`
- 삭제 영역: 더 명확한 Danger Zone 스타일링 (bg-red-50 배경)
