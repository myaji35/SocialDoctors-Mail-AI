# Pulse Marketing Agent - Knowledge Base

This directory contains the learning and reference materials that Pulse uses to generate on-brand, effective marketing content.

## Directory Structure

### `/services/`
Information about each of the 10+ SaaS services in the SocialDoctors marketplace.

**Each service file should include:**
- Service name and tagline
- Core features and benefits
- Target audience
- Key differentiators
- Pricing information
- Success stories/testimonials
- Messaging frameworks that work

### `/partners/`
Profiles and preferences for partners, influencers, and affiliates.

**Each partner profile should include:**
- Partner name and platform(s)
- Audience demographics
- Content style and preferences
- Past collaboration history
- Successful content patterns
- Commission structure
- Contact information

### `/personas/`
Target audience personas for different customer segments.

**Each persona should include:**
- Demographic information
- Goals and motivations
- Pain points and challenges
- Content preferences
- Messaging that resonates
- Where they spend time online
- Decision-making factors

## How Pulse Uses This Knowledge

### Content Generation
Before creating any content, Pulse:
1. Loads `brand-guidelines.md` for voice and style
2. References relevant service information
3. Considers target persona characteristics
4. Checks partner preferences if applicable

### Learning Over Time
As campaigns are executed:
- Successful messaging gets documented in service files
- Partner preferences are refined based on feedback
- Personas are updated with new insights
- Brand guidelines evolve with market response

### Cross-Reference Intelligence
Pulse can:
- Recommend service combinations based on persona needs
- Match partners to services that fit their audience
- Suggest content angles that worked for similar contexts
- Identify gaps in knowledge that need filling

## Maintaining This Knowledge Base

### Regular Updates
- Add new services as they launch
- Update pricing and features as they change
- Add new partner profiles as partnerships form
- Refine personas based on user research

### Quality Standards
- Keep information current and accurate
- Use consistent formatting across files
- Document sources for claims and statistics
- Remove outdated information

### Privacy & Security
- Store sensitive data (API keys, passwords) elsewhere
- Partner contact info and agreements are confidential
- Respect GDPR and privacy regulations
- Only store what's necessary for marketing purposes

---

**Start Here:** If you're setting up Pulse for the first time, begin by:
1. Reviewing and customizing `brand-guidelines.md`
2. Adding your first service file to `/services/`
3. Creating your primary target persona in `/personas/`
4. (Optional) Adding partner profiles as partnerships develop
